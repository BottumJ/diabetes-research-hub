## Post 4: Reddit r/bioinformatics

**Title:** Open-source computational analysis of the diabetes research landscape — looking for feedback

I built an open-source research hub that maps the global diabetes research landscape using publicly available data (PubMed, ClinicalTrials.gov). Looking for feedback from the bioinformatics community.

**What it does:**
- Literature gap analysis: 30 domains queried against PubMed, 435 pairwise combinations scored for research gap size
- Clinical trial intelligence: 746 trials pulled from ClinicalTrials.gov API v2, categorized and visualized
- PubMed monitoring: 15 alert queries tracking recent publications with cross-domain paper detection
- All analysis via Python scripts (standard library only, no heavy dependencies)

**What I found:**
Some genuinely surprising gaps — zero joint publications between Autoimmunity and Gene Therapy for diabetes, zero between Beta Cell Regeneration and Health Equity. These feel like areas where even basic computational synthesis could add value.

**Methodology:**
Triple-source validation framework, PRISMA 2020 alignment, pre-registered on OSF. Full details in the Research Doctrine.

**What I'm looking for:**
- Methodological critique (especially the gap scoring formula: normalized inverse overlap using geometric mean)
- Suggestions for additional public datasets to integrate
- Feedback on the upcoming multi-omics biomarker integration pilot
